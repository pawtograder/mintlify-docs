# Pawtograder CLI

The Pawtograder CLI provides command-line tools for instructors and site admins to manage courses, assignments, rubrics, and student repositories.

## Installation

The CLI is included in the platform repository. No separate installation is required.

## Authentication

Before using the CLI, you need to authenticate with an API token.

### Creating an API Token

1. Log in to Pawtograder web app
2. Navigate to **User Menu → API Tokens**
3. Create a new token with type **"CLI"** or **"MCP + CLI"**
4. Copy the token (starts with `mcp_`)

### Login

```bash
npm run cli -- login
```

You'll be prompted to enter your API token. The token is stored securely in `~/.pawtograder/credentials.json`.

You can also provide the token directly:

```bash
npm run cli -- login --token mcp_your_token_here
```

### Check Current User

```bash
npm run cli -- whoami
```

### Logout

```bash
npm run cli -- logout
```

## Core Commands

### Classes

List and view class information.

```bash
# List all classes
npm run cli -- classes list

# Show class details
npm run cli -- classes show <class-id-or-slug>
```

### Assignments

Manage assignments across classes.

```bash
# List assignments for a class
npm run cli -- assignments list --class <class-id>

# Show assignment details
npm run cli -- assignments show <assignment-id> --class <class-id>

# Copy assignments between classes
npm run cli -- assignments copy \
  --source-class cs3500-fall-2025 \
  --target-class cs3500-spring-2026 \
  --all

# Copy with schedule (CSV file with dates)
npm run cli -- assignments copy \
  --source-class cs3500-fall-2025 \
  --target-class cs3500-spring-2026 \
  --schedule schedule.csv \
  --workdir ./repos

# Delete an assignment
npm run cli -- assignments delete <assignment-id> \
  --class <class-id> \
  --force
```

#### Assignment Copy Options

| Option | Description |
|--------|-------------|
| `--source-class` | Source class (ID, slug, or name) |
| `--target-class` | Target class (ID, slug, or name) |
| `--assignment` | Single assignment to copy (ID or slug) |
| `--schedule` | CSV file with assignment slugs/titles and date overrides |
| `--all` | Copy all assignments from source class |
| `--dry-run` | Preview changes without making them |
| `--skip-repos` | Skip git repository operations |
| `--skip-rubrics` | Skip rubric copying |
| `--skip-surveys` | Skip copying surveys linked to assignments |
| `--workdir` | Local directory for git clones (required unless `--skip-repos`) |
| `--concurrency` | Parallel clone/fetch operations (1-8, default: 4) |
| `--delay-ms` | Delay between clone batches (ms) |

### Rubrics

Import and export rubrics in YAML format.

```bash
# List rubrics for an assignment
npm run cli -- rubrics list \
  --assignment <assignment-id> \
  --class <class-id>

# Export rubric to YAML
npm run cli -- rubrics export \
  --assignment <assignment-id> \
  --class <class-id> \
  --type grading \
  --output rubric.yml

# Import rubric from YAML
npm run cli -- rubrics import \
  --assignment <assignment-id> \
  --class <class-id> \
  --file rubric.yml \
  --type grading \
  --dry-run
```

Rubric types: `grading`, `self_review`, `meta`

### Surveys

Copy surveys between classes.

```bash
# Copy all surveys
npm run cli -- surveys copy \
  --source-class cs3500-fall-2025 \
  --target-class cs3500-spring-2026 \
  --all

# Copy specific survey
npm run cli -- surveys copy \
  --source-class cs3500-fall-2025 \
  --target-class cs3500-spring-2026 \
  --survey "Course Feedback" \
  --target-assignment hw1
```

### Flashcards

Copy flashcard decks between classes.

```bash
# List flashcard decks
npm run cli -- flashcards list --class <class-id>

# Copy all decks
npm run cli -- flashcards copy \
  --source-class cs3500-fall-2025 \
  --target-class cs3500-spring-2026 \
  --all
```

### Submissions

Batch import submission comments and artifacts.

```bash
# Import submission comments
npm run cli -- submissions comments import \
  --file batch-results.json \
  --class <class-id> \
  --assignment <assignment-id> \
  --author-profile-id <uuid>

# Sync submission comments (with soft-delete)
npm run cli -- submissions comments sync \
  --file batch-results.json \
  --class <class-id> \
  --assignment <assignment-id> \
  --rubric-part-id <id>

# Import submission artifacts
npm run cli -- submissions artifacts import \
  --file manifest.json \
  --class <class-id> \
  --assignment <assignment-id> \
  --batch-size 5
```

## Repository Maintenance

The CLI provides local git/rsync workflows for managing student repositories. These commands fetch metadata from the API and perform git operations on your machine using SSH.

<Warning>
Repository commands require SSH access to GitHub. Ensure your SSH keys are configured and you have access to the course organization.
</Warning>

### List Repositories

```bash
npm run cli -- repos list \
  --class <class-id> \
  --assignment <assignment-id>
```

### Sync Grade Workflow

Synchronize `.github/workflows/grade.yml` from the handout repository to all student repositories.

```bash
npm run cli -- repos sync-grade-workflow \
  --class <class-id> \
  --assignment <assignment-id> \
  --workdir ./repos \
  --concurrency 2
```

This command:
1. Fetches the canonical `grade.yml` from the handout repository
2. Clones all student repositories (or updates existing clones)
3. Compares the workflow file in each student repo
4. Commits and pushes updates where the file differs

Use `--dry-run` to preview changes without committing.

### Cross-Assignment Repository Copy

Copy files from source assignment repositories to target assignment repositories after the source due date.

```bash
npm run cli -- repos copy-after-source-due \
  --class <class-id> \
  --source-assignment hw1 \
  --target-assignment hw2 \
  --workdir ./repos \
  --concurrency 2
```

This command:
1. Verifies the source assignment due date has passed
2. Clones source and target repositories for each student
3. Uses `rsync` to copy files (excluding `.git`)
4. Commits and pushes changes to target repositories

Options:
- `--dry-run`: Preview with `rsync -n` only
- `--mirror-delete`: Pass `rsync --delete` (still excludes `.git`)
- `--concurrency`: Parallel operations (1-8, default: 2)
- `--delay-ms`: Delay between batches

### Convenience Alias

```bash
# Shorthand for repos commands
npm run cli:repos -- list --class <class-id> --assignment <assignment-id>
```

## Environment Variables

### HTTP Timeout

For large operations (e.g., artifact imports), increase the timeout:

```bash
export PAWTOGRADER_HTTP_TIMEOUT_MS=600000  # 10 minutes
```

### Debug Logging

Enable verbose HTTP logging:

```bash
export DEBUG=1
npm run cli -- <command>
```

## Common Workflows

### Semester Rollover

Copy an entire course to a new semester:

```bash
# 1. Copy all assignments with new dates
npm run cli -- assignments copy \
  --source-class cs3500-fall-2025 \
  --target-class cs3500-spring-2026 \
  --schedule spring-2026-schedule.csv \
  --workdir ./repos

# 2. Copy surveys
npm run cli -- surveys copy \
  --source-class cs3500-fall-2025 \
  --target-class cs3500-spring-2026 \
  --all

# 3. Copy flashcards
npm run cli -- flashcards copy \
  --source-class cs3500-fall-2025 \
  --target-class cs3500-spring-2026 \
  --all
```

### Bulk Rubric Updates

```bash
# 1. Export current rubric
npm run cli -- rubrics export \
  --assignment hw1 \
  --class cs3500-spring-2026 \
  --output hw1-rubric.yml

# 2. Edit hw1-rubric.yml locally

# 3. Preview changes
npm run cli -- rubrics import \
  --assignment hw1 \
  --class cs3500-spring-2026 \
  --file hw1-rubric.yml \
  --dry-run

# 4. Apply changes
npm run cli -- rubrics import \
  --assignment hw1 \
  --class cs3500-spring-2026 \
  --file hw1-rubric.yml
```

### Repository Maintenance

```bash
# Update grade workflows across all student repos
npm run cli:repos -- sync-grade-workflow \
  --class cs3500-spring-2026 \
  --assignment hw1 \
  --workdir ./repos \
  --dry-run

  --class cs3500-spring-2026 \
  --source-assignment hw1 \

```

## Error Handling

The CLI provides detailed error messages:

- **Authentication errors**: Token expired or invalid scopes
- **Permission errors**: Insufficient access to class or assignment
- **Timeout errors**: Operation took too long (increase `PAWTOGRADER_HTTP_TIMEOUT_MS`)
- **Validation errors**: Invalid input or missing required fields

Most commands support `--dry-run` to preview changes before applying them.

## Troubleshooting

### Connection Timeouts

Assignment copy operations can take several minutes. If you encounter 504 Gateway Timeout errors:

1. The operation may have partially completed
2. Run the command again (it validates and fixes existing assignments)
3. Set `DEBUG=1` for detailed logging

### Repository Clone Failures

If SSH git operations fail:

1. Verify SSH keys are configured: `ssh -T git@github.com`
2. Check GitHub organization access
3. Retry with lower `--concurrency` (default: 4 → try 2 or 1)
4. Use `--delay-ms 1000` to add delays between batches

### Large Artifact Imports

For large base64 payloads:

1. Reduce `--batch-size` (default: 5 → try 1)
2. Increase timeout: `export PAWTOGRADER_HTTP_TIMEOUT_MS=600000`
3. Split the manifest into smaller files

## API Token Scopes

CLI tokens require specific scopes:

- `cli:read`: Read-only operations (list, show, export)
- `cli:write`: Write operations (copy, import, delete)

Create tokens with appropriate scopes in the web UI under **Settings → API Tokens**.
