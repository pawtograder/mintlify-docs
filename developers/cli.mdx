## Installation

The CLI is included in the platform repository. Run it using npm:

```bash
npm run cli -- <command> [options]
```

For repository commands, use the shortcut:

```bash
npm run cli:repos -- <subcommand> [options]
```

## Authentication

Before using the CLI, you need to authenticate with an API token.

### Creating an API Token

1. Log in to Pawtograder web app
2. Navigate to **User Menu → API Tokens**
3. Create a new token with type **"CLI"** or **"MCP + CLI"**
4. Copy the token (starts with `mcp_`)

2. Go to User Menu → API Tokens
3. Click "Create New Token"
4. Select "CLI" or "MCP + CLI" as the token type

#### Assignment Copy Options

- `--source-class`: Source class identifier (ID, slug, or name)
- `--target-class`: Target class identifier (ID, slug, or name)
- `--assignment`: Copy a single assignment by ID or slug
- `--schedule`: CSV file with assignment slugs/titles and date overrides
- `--all`: Copy all assignments from source class
- `--workdir`: Local directory for git repository operations (required unless `--skip-repos`)
- `--dry-run`: Preview what would be copied without making changes
- `--skip-repos`: Skip git repository operations
- `--skip-rubrics`: Skip rubric copying
- `--skip-surveys`: Skip copying surveys linked to assignments
- `--concurrency`: Number of parallel git operations (1-8, default: 4)
- `--delay-ms`: Delay between clone batches in milliseconds (default: 0)
- `--debug`: Enable server-side timing logs

## Troubleshooting

### Authentication errors

If you encounter authentication errors:

1. Verify your API token is valid
2. Check that you're logged in: `npm run cli -- whoami`
3. Try logging out and back in: `npm run cli -- logout && npm run cli -- login`

### Command not found

Ensure you're running the CLI from the platform repository root directory and have installed dependencies with `npm install`.

### Permission errors

Some commands require specific permissions. Ensure your user account has the necessary role (instructor or admin) for the operation you're attempting.

If you see "Not logged in" errors:
1. Run `npm run cli -- login` to authenticate
2. Verify your token has the required scopes (cli:read or cli:write)
3. Check that your token hasn't been revoked

### Timeout errors

For large operations (copying many assignments or repos):

## Error Handling

The CLI provides detailed error messages and logging. Use the `--debug` flag on supported commands to enable verbose server-side logging.

If you encounter authentication errors, try logging out and logging back in:

```bash
npm run cli -- logout
npm run cli -- login
```

The CLI provides detailed error messages:

- **Authentication errors**: Token expired or invalid scopes
- **Permission errors**: Insufficient access to class or assignment
- **Timeout errors**: Operation took too long (increase `PAWTOGRADER_HTTP_TIMEOUT_MS`)
- **Validation errors**: Invalid input or missing required fields

Most commands support `--dry-run` to preview changes before applying them.

## Troubleshooting

### Connection Timeouts

Assignment copy operations can take several minutes. If you encounter 504 Gateway Timeout errors:

1. The operation may have partially completed
2. Run the command again (it validates and fixes existing assignments)
3. Set `DEBUG=1` for detailed logging

### Repository Clone Failures

If SSH git operations fail:

1. Verify SSH keys are configured: `ssh -T git@github.com`
2. Check GitHub organization access
3. Retry with lower `--concurrency` (default: 4 → try 2 or 1)
4. Use `--delay-ms 1000` to add delays between batches

### Large Artifact Imports

For large base64 payloads:

1. Reduce `--batch-size` (default: 5 → try 1)
2. Increase timeout: `export PAWTOGRADER_HTTP_TIMEOUT_MS=600000`
3. Split the manifest into smaller files

## API Token Scopes

CLI tokens require specific scopes:

- `cli:read`: Read-only operations (list, show, export)
- `cli:write`: Write operations (copy, import, delete)

Create tokens with appropriate scopes in the web UI under **Settings → API Tokens**.

- **401 Unauthorized**: Token expired or revoked - run `pawtograder login`
- **403 Forbidden**: Token lacks required scopes (cli:read or cli:write)
- **504 Gateway Timeout**: Operation took too long - increase timeout or use smaller batches

For assignment copy operations, the CLI validates and fixes existing assignments on retry, so you can safely re-run failed commands.

## Best Practices

1. **Use dry-run first**: Preview changes with `--dry-run` before executing
2. **Export backups**: Export rubrics before importing new versions
3. **Batch size**: For artifact imports, use smaller `--batch-size` if requests timeout
4. **Workdir**: Use a dedicated directory for `--workdir` to avoid conflicts
5. **Concurrency**: Start with default concurrency and adjust based on network performance

## Troubleshooting

### Authentication Errors

If you see "Not logged in" or "Authentication failed":

1. Run `npm run cli -- login` to re-authenticate
2. Verify your token is valid in the web UI (Settings → API Tokens)
3. Ensure your token has the correct scopes (`cli:read` or `cli:write`)

### Repository Operations Failing

Repository commands use SSH git authentication. Ensure:

1. You have SSH keys configured for GitHub
2. Your SSH key is added to your GitHub account
3. You can clone repositories manually: `git clone git@github.com:org/repo.git`

### Timeout Errors

For large operations (copying many assignments, importing large artifacts):

1. Set a longer timeout: `PAWTOGRADER_HTTP_TIMEOUT_MS=600000 npm run cli -- ...`
2. Use smaller batch sizes: `--batch-size 1` for artifact imports
3. Use `--dry-run` first to validate before running the full operation

### Connection Reset Errors

The CLI automatically retries git operations that fail due to connection resets. If you see persistent failures:

1. Check your network connection
2. Verify GitHub is accessible
3. Try reducing `--concurrency` to avoid rate limits
