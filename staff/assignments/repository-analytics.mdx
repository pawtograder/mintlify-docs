# Repository Analytics

Pawtograder provides analytics for student assignment repositories, allowing you to track commit activity, repository engagement, and development patterns across your course.

## Viewing Repository Analytics

Navigate to an assignment's analytics page to see repository statistics for all student submissions.

### Available Metrics

The repository analytics dashboard displays:

- **Commit count**: Total number of commits made to each student's repository
- **Commit timeline**: Visual representation of when commits were made
- **Active contributors**: Number of unique contributors to group repositories
- **Repository activity**: Last commit timestamp and activity patterns
- **Commit frequency**: Distribution of commits over time

## Understanding Commit Patterns

### Individual Assignments

For individual assignments, the analytics show:

- Single student's commit history
- Commit frequency over the assignment period
- Timestamps of first and last commits
- Total commit count

### Group Assignments

For group assignments, the analytics include:

- Combined commit history from all group members
- Individual contribution breakdown by student
- Commit distribution among group members
- Group collaboration patterns

## Using Analytics for Assessment

Repository analytics can help you:

- **Identify early starters vs. procrastinators**: See when students begin working on assignments
- **Detect collaboration issues**: In group projects, identify uneven contribution patterns
- **Monitor progress**: Track whether students are making steady progress or cramming before deadlines
- **Verify academic integrity**: Unusual commit patterns may warrant further investigation
- **Provide targeted support**: Reach out to students with low or no commit activity

<Warning>
Commit count alone is not a reliable measure of effort or quality. Some students make many small commits while others make fewer, larger commits. Use analytics as one data point among many when assessing student work.
</Warning>

## Exporting Analytics Data

Click **Export to CSV** to download repository analytics for external analysis or record-keeping.

The CSV export includes:

- Student name and email
- Repository URL
- Total commit count
- First commit timestamp
- Last commit timestamp
- Commit frequency metrics

<Tip>
Combine repository analytics with gradebook data and submission timestamps to get a complete picture of student engagement and progress.
</Tip>

## Privacy and Access

- Only instructors and graders can view repository analytics
- Students cannot see analytics for other students' repositories
- Analytics data is derived from GitHub repository metadata
- No code content is analyzed or stored, only commit metadata

## Enabling Repository Analytics

To enable repository analytics for an assignment:

1. Navigate to the assignment configuration page
2. Check the **Enable repository analytics** option
3. Save the assignment

Repository analytics are only available for assignments with GitHub repositories enabled.

## Linking Rubric Checks to Analytics

You can link rubric checks to specific repository analytics categories. When grading, these links allow you to quickly view relevant repository activity.

To link a rubric check to analytics, add a `kpi_category` field to the check in your rubric YAML:

```yaml
checks:
  - name: Made meaningful commits
    points: 10
    kpi_category: commits
  - name: Participated in code review
    points: 5
    kpi_category: pr_review_comments
```

Supported KPI categories:
- `commits`: Links to commit history
- `prs_opened`: Links to pull requests
- `pr_review_comments`: Links to PR review comments
- `issues_opened`: Links to issues opened
- `issues_closed`: Links to issues closed
- `issue_comments`: Links to issue comments

When grading with linked checks, a "View analytics" link appears next to the check, filtering the analytics view to show only relevant items.

## Use Cases

### Individual Contribution Auditing

For group assignments, use repository analytics to verify that all team members are contributing:

- Compare commit counts across group members
- Review PR authorship and review participation
- Identify students who may need additional support

### Process Compliance

Verify students are following required development practices:

- Check that students are creating PRs for code review
- Ensure issues are being used for task tracking

## Linking Rubric Checks to Analytics

Rubric checks can be linked to specific repository analytics KPIs (Key Performance Indicators). When configuring a rubric check, you can assign a KPI category:

- `commits`: Track commit activity
- `prs_opened`: Track pull requests created
- `pr_review_comments`: Track code review participation
- `issues_opened`: Track issues created
- `issues_closed`: Track issue resolution
- `issue_comments`: Track issue discussion participation

When a check has a KPI category assigned, graders see a "View analytics" link that filters the repository analytics to show only that activity type.

## How It Works

Repository analytics uses the GitHub API to fetch activity data:

1. **Data Collection**: The system fetches issues, pull requests, commits, and comments from the assignment repository
2. **Incremental Updates**: After the initial fetch, subsequent updates only retrieve new activity since the last fetch
3. **Daily Aggregation**: Activity is aggregated by day for the activity chart
4. **Per-Submission View**: Each submission shows analytics for its associated repository

### Rate Limiting

To respect GitHub API rate limits:
- Manual refreshes are limited to once per 10 minutes per repository
- Automatic daily refreshes are staggered across the 2:00 AM UTC hour
- The system monitors GitHub rate limits and delays processing when necessary
- Bulk operations process repositories in batches of 20

## Privacy and Access

- Repository analytics are only visible to instructors and graders
- Students cannot view repository analytics
- Analytics data is scoped to the assignment's repository
- All data is fetched using the course's configured GitHub organization
