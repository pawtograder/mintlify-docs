# Assignment Security Dashboard

The assignment security dashboard helps you monitor and investigate potential academic integrity issues by tracking security-related events across student submissions.

## Accessing the Dashboard

Navigate to an assignment and click the **Security** tab to view the security dashboard.

## Security Events Tracked

The dashboard monitors various security-related events, including:

- **Unusual submission patterns**: Multiple submissions in rapid succession
- **Repository access**: Who accessed student repositories and when
- **File modifications**: Timestamps and patterns of file changes
- **Collaboration indicators**: Similarities between submissions
- **External access attempts**: Unauthorized access attempts to student repositories

## Dashboard Components

### Event Timeline
View a chronological timeline of security events for the assignment:
- Event type and description
- Student(s) involved
- Timestamp
- Severity level (info, warning, critical)

### Student Activity Summary
For each student, see:
- Total number of commits
- Submission timestamps
- Repository access logs
- Flagged events

### Similarity Analysis
The dashboard can identify potential collaboration issues by analyzing:
- Code similarity between submissions
- Shared commit patterns
- Identical file structures or content

## Investigating Events

Click on any security event to view detailed information:

<Steps>
<Step title="Select Event">
Click on a flagged event in the timeline
</Step>

<Step title="Review Details">
View comprehensive information including:
- Full event description
- Students involved
- Related submissions
- Repository access logs
</Step>

<Step title="Compare Submissions">
If the event involves multiple students, you can open their submissions side-by-side for comparison
</Step>

<Step title="Take Action">
Based on your investigation:
- Add notes to the event for future reference
- Flag submissions for manual review
- Contact students for clarification
- Report to academic integrity office if needed
</Step>
</Steps>

## Filtering and Search

Use the dashboard filters to:
- Show only high-severity events
- Filter by specific students or groups
- View events within a date range
- Search for specific event types

## Privacy and Compliance

The security dashboard is designed with student privacy in mind:
- All monitoring is limited to course-related repositories
- Access logs are retained according to your institution's policies
- Only instructors and designated staff can view security events
- Students are informed about monitoring through course policies

## Best Practices

- **Review regularly**: Check the dashboard after each assignment deadline
- **Investigate promptly**: Address flagged events quickly while details are fresh
- **Document findings**: Add notes to events for future reference
- **Communicate policies**: Ensure students understand academic integrity expectations
- **Use as educational tool**: Discuss common issues with students to prevent future violations
