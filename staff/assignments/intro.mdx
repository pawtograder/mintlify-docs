# Assignments Overview

Programming assignments are the core of Pawtograder. Each assignment consists of a "handout" repository, and optionally, a "grader" repository. When an assignment is released, Pawtograder automatically creates a new private repository for each student based on that handout. Pawtograder automatically manages permissions for all of these repositories, allowing staff to view all of a course's repositories, and ensuring students can only view their own repositories.

## GitHub Permissions

Pawtograder creates two GitHub teams for each class: a "staff" team and a "students" team. Staff are granted access to all class repositories, and students are only granted access to their own repositories. The "students" team can be used by instructors who want to create additional GitHub repositories that are only accessible to students (e.g. for distributing in-class exercises).

### Repository Permissions Audit

Instructors can audit repository permissions to ensure that all students and staff have the correct access to assignment repositories. This is particularly useful for:

- Verifying that students have access to their repositories after group changes
- Ensuring staff permissions are correctly configured
- Troubleshooting access issues
- Maintaining security by confirming no unauthorized access exists

**To run a permissions audit:**

1. Navigate to the assignment's settings page
2. Click on "Repository Permissions Audit"
3. Review the audit results showing current permissions for all repositories
4. Identify any discrepancies or missing permissions
5. Use the "Fix Permissions" button to automatically correct any issues

The audit flow helps maintain the integrity of repository access, especially important when:
- Students are added to or removed from groups
- Group assignments are modified after release
- Staff roles change during the semester
- Troubleshooting student access problems

## Repositories

Two repositories are automatically created for each assignment: a "handout" repository and a "grader" repository. These repositories must be configured by the staff before the assignment is released.

A **handout repository** is a repository that contains the starter code for an assignment. It is used to provide students with the necessary files to complete the assignment.

A **grader repository** is a repository that contains the code that will be used to grade the assignment. It is used to provide instructors with the necessary files to grade the assignment.

## Individual and Group Assignments

Assignments can be individual, group-optional, or group-required.
Students can be assigned to groups by instructors, or can self-assign to groups.
Instructors can reuse a set of groups from a prior assignment.

### Improved Group Management

When tweaking student groups after an assignment is released, Pawtograder provides improved resilience:

- Group changes are processed more reliably
- Repository permissions are automatically updated when groups change
- Students are only notified of group assignments after the assignment is released (not during draft configuration)
- Prevents premature notifications that could confuse students before an assignment goes live

## Assignment Dashboard and Metrics

The instructor dashboard for each assignment provides comprehensive metrics and insights into student progress and grading status:

### Key Metrics Displayed

- **Submission Statistics**: Total submissions, on-time vs late submissions, and submission rate
- **Grading Progress**: Number of submissions graded, pending reviews, and completion percentage
- **Student Engagement**: Active students, group formation status, and repository activity
- **Autograder Results**: Pass rates, common failures, and test execution statistics

### Assignment Overview

The revamped assignment overview page provides instructors with:

- **At-a-glance status**: Quick view of assignment health and progress
- **Actionable insights**: Identify students who need help or follow-up
- **Grading workflow**: Direct access to pending grading tasks and review assignments
- **Timeline visualization**: See submission patterns and deadline adherence

These metrics help instructors monitor assignment progress, identify issues early, and make data-driven decisions about extensions, additional support, or grading priorities.

## Due Date Exceptions

Instructors can set a class-wide number of late tokens, and configure assignments to allow students to use those tokens as desired.
Each token can be exchanged for a 24-hour extension, but the extension must be requested before the student's original due date.

Instructors and graders can also manually grant extensions to students, and can view all extensions granted to a student. This is done in the assignment configuration under "Due Date Exceptions".
