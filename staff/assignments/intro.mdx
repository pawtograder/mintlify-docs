# Assignments Overview

Programming assignments are the core of Pawtograder. Each assignment consists of a "handout" repository, and optionally, a "grader" repository. When an assignment is released, Pawtograder automatically creates a new private repository for each student based on that handout. Pawtograder automatically manages permissions for all of these repositories, allowing staff to view all of a course's repositories, and ensuring students can only view their own repositories.

## GitHub Permissions

Pawtograder creates two GitHub teams for each class: a "staff" team and a "students" team. Staff are granted access to all class repositories, and students are only granted access to their own repositories. The "students" team can be used by instructors who want to create additional GitHub repositories that are only accessible to students (e.g. for distributing in-class exercises).

## Repositories

Two repositories are automatically created for each assignment: a "handout" repository and a "grader" repository. These repositories must be configured by the staff before the assignment is released.

A **handout repository** is a repository that contains the starter code for an assignment. It is used to provide students with the necessary files to complete the assignment.

A **grader repository** is a repository that contains the code that will be used to grade the assignment. It is used to provide instructors with the necessary files to grade the assignment.

### Syncing Handout Updates

When you need to push updates from the handout repository to student repositories (for example, to fix a bug in starter code), Pawtograder provides a sync-to-handout feature. The system has been improved for better resilience:

- **Automatic retry logic**: Failed syncs are automatically retried
- **Conflict detection**: The system detects and reports merge conflicts
- **Selective sync**: Choose which students receive the update
- **Status tracking**: Monitor sync progress across all student repositories

This ensures that updates reach all students reliably, even in cases of temporary network issues or repository conflicts.

After an assignment is released, you may need to update the handout repository to fix bugs or add clarifications. Pawtograder provides a sync-to-handout feature that:

- Pushes updates from the handout repository to all student repositories
- Includes improved resilience to handle edge cases and errors
- Preserves student work while merging in your changes
- Notifies students of the updates

This feature ensures all students receive important fixes and updates without disrupting their work in progress.

## Empty Submission Detection

Pawtograder automatically detects when students submit code that matches the handout (starter) files exactly. This helps identify students who may have forgotten to commit their changes before submitting.

When creating an assignment, you can choose whether to:

- **Permit empty submissions** (default): Students can submit even if their files match the handout exactly. The submission is flagged as empty but accepted.
- **Reject empty submissions**: Students receive an error message asking them to commit their changes. The submission is automatically removed.

<Note>
Instructors and graders can always submit, even when empty submissions are prohibited. This prevents breaking staff testing workflows.
</Note>

Empty submission detection works by:
1. Computing SHA-256 hashes of all expected submission files from the handout repository
2. Comparing submitted files against all recorded handout versions
3. Flagging submissions that match any handout version

### Managing Student Groups

The group management interface allows instructors to:

- Create and modify student groups
- Move students between groups
- Adjust group sizes and composition
- View group formation status

**Important**: Students are not notified about group assignments until the assignment is released. This allows instructors to:
- Prepare groups in advance without alerting students
- Make adjustments before the assignment goes live
- Avoid confusion from premature notifications

The group management UI is designed to be resilient, allowing you to make changes even when there are temporary issues with repository creation or permissions.

### Improved Group Management

When tweaking student groups after an assignment is released, Pawtograder provides improved resilience:

- Group changes are processed more reliably
- Repository permissions are automatically updated when groups change
- Students are only notified of group assignments after the assignment is released (not during draft configuration)
- Prevents premature notifications that could confuse students before an assignment goes live

## Due Date Exceptions

Instructors can set a class-wide number of late tokens, and configure assignments to allow students to use those tokens as desired.
Each token can be exchanged for a 24-hour extension, but the extension must be requested before the student's original due date.

Instructors and graders can also manually grant extensions to students, and can view all extensions granted to a student. This is done in the assignment configuration under "Due Date Exceptions".

## Assignment Score Capping

Instructors can configure a maximum score cap for assignments. Score capping allows you to:

- Set an upper limit on the total points a student can earn
- Prevent extra credit from inflating scores beyond a desired maximum
- Ensure consistency in grading across different rubric configurations
- Cap scores at 100% even if bonus points are available

### Configuring Score Caps

To set a score cap for an assignment:

1. Navigate to the assignment configuration
2. Find the **Score Cap** settings
3. Enter the maximum score (e.g., 100 for a 100-point assignment)
4. Save the configuration

When a score cap is set:
- Student scores are automatically capped at the specified maximum
- The gradebook displays the capped score
- Students can see their raw score and the capped score on their submission page

<Tip>
Use score capping when you offer bonus points or extra credit but want to ensure no student exceeds the assignment's maximum possible score.
</Tip>
