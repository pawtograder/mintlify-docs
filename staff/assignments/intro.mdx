# Assignments Overview

Programming assignments are the core of Pawtograder. Each assignment consists of a "handout" repository, and optionally, a "grader" repository. When an assignment is released, Pawtograder automatically creates a new private repository for each student based on that handout. Pawtograder automatically manages permissions for all of these repositories, allowing staff to view all of a course's repositories, and ensuring students can only view their own repositories.

## GitHub Permissions

Pawtograder creates two GitHub teams for each class: a "staff" team and a "students" team. Staff are granted access to all class repositories, and students are only granted access to their own repositories. The "students" team can be used by instructors who want to create additional GitHub repositories that are only accessible to students (e.g. for distributing in-class exercises).

### Repository Permissions Audit

Instructors can audit repository permissions to ensure students have correct access to their assignment repositories. The audit flow helps identify and fix permission issues that may prevent students from accessing their work.

To run a permissions audit:

1. Navigate to the assignment's **Repositories** page (or the assignment's settings page).
2. Click **Fix Repository Permissions** (also called **Audit Repository Permissions**).
3. Review the audit report, which shows:
   - Students with correct repository access
   - Students missing repository access
   - Students with incorrect permission levels
   - Orphaned permissions from dropped students
4. Click **Fix All Issues** to automatically correct problems, or click **Fix** next to individual students to correct specific issues.
5. View the summary showing synced repositories, skipped repositories, and any errors.

The audit validates that all group members have correct access, grants staff access to all repositories, and removes access for students who have left groups or the course.

Run the audit periodically, especially after:

- Adding or removing students from the course
- Changing group assignments
- Students reporting access issues
- Manual changes to GitHub teams

<Note>
Run the permissions audit if students report being unable to access their repositories or if you've made manual changes to GitHub teams.
</Note>

## Repositories

Two repositories are automatically created for each assignment: a "handout" repository and a "grader" repository. These repositories must be configured by the staff before the assignment is released.

A **handout repository** contains the starter code for an assignment. It provides students with the necessary files to complete the assignment.

A **grader repository** contains the code that will be used to grade the assignment. It provides instructors with the necessary files to grade the assignment.

### Syncing Handout Updates

After an assignment is released, you may need to update the handout repository to fix bugs or add clarifications. Pawtograder provides a sync-to-handout feature that pushes updates from the handout repository to all student repositories while preserving student work. Key capabilities include:

- **Automatic retry logic**: Failed syncs are automatically retried.
- **Conflict detection**: The system detects and reports merge conflicts.
- **Selective sync**: Choose which students receive the update.
- **Status tracking**: Monitor sync progress across all student repositories.

This ensures that updates reach all students reliably, even in cases of temporary network issues or repository conflicts.

### Repository Analytics

Pawtograder provides analytics for assignment repositories to help instructors track student activity and engagement. Repository analytics include:

- Commit frequency and patterns across all student repositories
- Push activity and timing relative to assignment deadlines
- Lines of code added and removed
- File modification patterns and contribution graphs over time
- Repository creation and setup completion rates
- Collaboration metrics for group assignments

Access repository analytics from the assignment page to identify students who may need additional support or who are struggling to get started.

## Individual and Group Assignments

Assignments can be individual, group-optional, or group-required.
Students can be assigned to groups by instructors, or can self-assign to groups.
Instructors can reuse a set of groups from a prior assignment.

### Managing Student Groups

Instructors manage groups through the **Groups** page (also called "Configure Groups") for each assignment. The interface provides:

- **Create groups manually**: Form new groups and assign initial members.
- **Edit existing groups**: Add or remove students, rename groups, or move students between groups.
- **Delete groups**: Remove groups that are no longer needed.
- **Bulk operations**: Create multiple groups at once, import from CSV, or copy groups from previous assignments.
- **Inline editing**: Update memberships, assign mentors, and rename groups directly in the table.
- **Publish changes**: Stage multiple changes locally, then publish them together to apply atomic updates.

When you publish group changes, Pawtograder automatically:

1. Creates or updates group memberships in the database.
2. Creates GitHub repositories for new groups (when the assignment has a template repository).
3. Queues GitHub permission synchronization tasks asynchronously.

Students are not notified about group assignments until the assignment is released, so you can prepare and adjust groups in advance without alerting students.

This workflow is particularly useful for:

- Pre-assigning students to groups based on sections or preferences
- Managing group assignments for large classes
- Ensuring balanced group sizes
- Quickly reorganizing groups before an assignment begins

### Group Mentors

Instructors can assign mentors to student groups from the assignment's **Groups** page. Group mentors:

- Are displayed to students in their group information
- Can be automatically assigned to grade their group's submissions using the "Assign to group mentors" bulk grading mode
- Appear in the groups CSV export for tracking purposes
- Are preserved when copying groups between assignments

To assign a mentor, use the **Mentor** dropdown for each group and select a grader or instructor.

### Group Individual Grading

For group assignments, instructors can enable individual grading to assess each group member's contribution separately. This allows:

- Assigning different grades to different members of the same group
- Tracking individual performance within collaborative projects
- Providing personalized feedback to each group member

Individual grades can be set through the gradebook interface, overriding the default group grade for specific students. This is useful for peer evaluation or when group members contribute unequally to the project.

### Exporting Group and Student Data

You can export group and student information to CSV for external analysis or record-keeping:

- **Group export**: Group name, student names with emails (in `Name <email>` format), and mentor name with email (if assigned).
- **Student export**: Student information including email addresses in the same `Name <email>` format.

Exported CSV files use UTF-8 encoding with BOM for Excel compatibility, include formula-safe escaping to prevent injection attacks, and are sorted alphabetically for consistent exports.

## Empty Submission Detection

Pawtograder automatically detects when students submit code that matches the handout (starter) files exactly. This helps identify students who may have forgotten to commit their changes before submitting.

When creating an assignment, you can choose whether to:

- **Permit empty submissions** (default): Students can submit even if their files match the handout exactly. The submission is flagged as empty but accepted.
- **Reject empty submissions**: Students receive an error message asking them to commit their changes. The submission is automatically removed.

<Note>
Instructors and graders can always submit, even when empty submissions are prohibited. This prevents breaking staff testing workflows.
</Note>

Empty submission detection works by:

1. Computing SHA-256 hashes of all expected submission files from the handout repository.
2. Comparing submitted files against all recorded handout versions.
3. Flagging submissions that match any handout version.

## Due Date Exceptions

Instructors can set a class-wide number of late tokens, and configure assignments to allow students to use those tokens as desired.
Each token can be exchanged for a 24-hour extension, but the extension must be requested before the student's original due date.

Instructors and graders can also manually grant extensions to students, gift additional late tokens to individual students, and view all extensions granted to a student. This is done in the assignment configuration under **Due Date Exceptions**. For group assignments, due date exceptions are applied at the group level so all group members receive the same extension.

### Granting Manual Extensions

When granting manual extensions, instructors choose the extended due date directly using a date picker, rather than entering extension hours and minutes. This makes it easier to:

- Set specific deadline times (e.g., "Friday at 5:00 PM")
- Grant extensions that don't align with 24-hour increments
- Visualize the new deadline while setting the extension
- Avoid calculation errors when determining extension length

The date picker shows the student's current due date and allows you to select any future date and time for the extended deadline.

### Gifting Late Tokens

Instructors can gift additional late tokens to individual students beyond the class-wide allocation. Gifted tokens:

- Are added to the student's available token balance
- Can be used just like regular late tokens
- Display correctly in the student's remaining token count
- Allow students who have exhausted their initial tokens to still request extensions

This is useful for accommodating special circumstances or rewarding exceptional participation.

### Self-Review Deadlines

When an assignment's due date is extended (either through late tokens or instructor exceptions), self-review deadlines are automatically adjusted to maintain the appropriate time window for students to complete their self-reviews.

## Time Zone Support

Pawtograder supports configurable time zones for displaying assignment deadlines and other time-sensitive information. Instructors can set a default time zone for the course, and individual users can configure their preferred time zone in their settings. This ensures that all deadlines are displayed accurately regardless of where students and staff are located.

## Score Capping

Instructors can configure a maximum score cap for assignments. Score capping allows you to:

- Set an upper limit on the total points a student can earn
- Prevent extra credit from inflating scores beyond a desired maximum
- Ensure consistency in grading across different rubric configurations
- Cap scores at 100% even if bonus points are available

To set a score cap:

1. Navigate to the assignment configuration.
2. Find the **Score Cap** settings.
3. Enter the maximum score (e.g., 100 for a 100-point assignment).
4. Save the configuration.

When a score cap is set, student scores are automatically capped at the specified maximum. The gradebook displays the capped score, and students can see both their raw score and the capped score on their submission page.

<Tip>
Use score capping when you offer bonus points or extra credit but want to ensure no student exceeds the assignment's maximum possible score.
</Tip>

## Leaderboard

Assignments can optionally include a leaderboard to display student rankings based on autograder scores. The leaderboard:

- Shows top-performing submissions and updates automatically as submissions are graded
- Can be enabled or disabled per assignment
- Supports a "less cluttering" display option for a more subtle appearance
- Encourages friendly competition when enabled

Toggle the leaderboard on or off in the assignment settings to match the pedagogical goals of each assignment.

<Tip>
Consider your course culture when deciding whether to enable this feature. The leaderboard can encourage engagement while remaining optional for courses that prefer not to emphasize rankings.
</Tip>

## Assignment Dashboards

Pawtograder provides several dashboards to help you monitor and manage assignments:

### Assignment Dashboard

The assignment overview page provides a comprehensive view of assignment progress and health. Key metrics include:

- **Submission statistics**: Total submissions, on-time vs. late submissions, and submission trends over time.
- **Grading progress**: Submissions graded, pending reviews, and grading completion percentage.
- **Student engagement**: Active students, repository activity, and group formation status.
- **Autograder results**: Pass rates, common failures, and test execution statistics.
- **Timeline visualization**: Submission patterns and deadline adherence.

Click into specific metrics for detailed breakdowns, use filters to focus on specific sections, groups, or graders, and export data for external analysis.

<Tip>
The dashboard updates in real-time as students submit work and graders complete reviews, giving you an up-to-date view of assignment progress.
</Tip>

### Grading Status Dashboard

The grading status dashboard provides a comprehensive overview of grading progress across all submissions:

- View completion status for each grading task
- Track which submissions have been graded and which are pending
- Monitor grader workload and distribution across TAs
- Identify bottlenecks in the grading workflow
- Complete multiple reviews at once with bulk actions

### Security Dashboard

The assignment security dashboard helps instructors monitor and detect potential academic integrity issues:

- **Anomaly detection**: Identify unusual submission patterns and timing.
- **Collaboration analysis**: Detect potential unauthorized collaboration.
- **Repository access monitoring**: Track who has accessed student repositories.
- **Submission history**: Review detailed commit and push patterns.
- **Flag potential issues**: Mark submissions for further review.

Access the security dashboard from the assignment page to review potential concerns and investigate suspicious activity while respecting student privacy.

### Test Insights Dashboard

The test insights dashboard provides detailed analytics on student performance across test cases:

- View aggregated test results across all submissions
- Identify common errors and failure patterns
- Track which test cases students struggle with most
- Export affected student emails for targeted communication
- Launch regrading workflows directly from error groups

Use the test insights dashboard to understand where students need additional support and to improve your assignments over time.
