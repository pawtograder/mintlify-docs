# Assignment Dashboard

The assignment dashboard provides a centralized view of all key metrics and information for an assignment. Use this dashboard to monitor student progress, track submissions, and manage assignment logistics and grading workflows.

## Accessing the Dashboard

Navigate to any assignment to view its dashboard — it is the default landing page for each assignment. For a course-wide view of all assignments, access the assignment dashboard from the course navigation.

## Dashboard Components

### Assignment overview

The top section displays essential assignment information:

- **Assignment name and description**
- **Due date and time**
- **Release status**: Whether the assignment is visible to students (Draft, Released, Due, Closed)
- **Submission count**: Number of students who have submitted

### Submission statistics

Track student progress with real-time statistics:

- **Total submissions**: Number of student submissions received
- **On-time vs. late**: Breakdown of submission timing
- **Missing submissions**: Students who haven't submitted yet
- **Submission rate**: Percentage of enrolled students who have submitted
- **Average score**: Current average across all graded submissions

### Grading progress

Monitor grading status across all submissions:

- **Graded**: Submissions with completed reviews
- **In progress**: Submissions currently being graded
- **Not started / pending**: Submissions awaiting grading assignment or review
- **Released grades**: Grades that have been released to students

#### Grader workload

View grading distribution across your staff:

- Which graders are assigned to which submissions
- Completion rates per grader
- Overdue grading tasks

### Student engagement metrics

Track how students are engaging with the assignment:

- **Repository activity**: Commit frequency and timing
- **Autograder runs**: How often students are testing their code
- **Help requests**: Office hours requests related to this assignment
- **Discussion posts**: Questions and discussions about the assignment

### Recent activity

See the latest updates and actions:

- Recent submissions
- Grading completions
- Student questions or issues
- System notifications

### Quick actions

Perform common assignment management tasks directly from the dashboard:

- **View all submissions**: Navigate to the submissions list.
- **Manage grading**: Assign grading tasks to staff.
- **Release grades**: Make graded submissions visible to students.
- **Edit assignment**: Modify assignment settings and configuration.
- **View test insights**: Access the test insights dashboard.
- **Check security**: Open the security dashboard.
- **Export data**: Download submission and grading data.

## Filtering and Sorting

Customize your dashboard view:

- Filter by submission status (submitted, pending, late) or by assignment status (draft, released, closed).
- Sort by score, submission time, student name, due date, or grading progress.
- Search for specific students or assignments.
- Group by section or team (for group assignments) or by assignment type.

## Visual Indicators

The dashboard uses visual cues to highlight important information:

- **Progress bars**: Show grading completion percentage.
- **Status badges**: Indicate assignment state (draft, active, closed).
- **Alert icons**: Flag assignments requiring attention (low submission rates, overdue grading).
- **Color coding**: Differentiate between on-time, late, and missing submissions.

## Using the Dashboard

### Monitoring student progress

1. Review submission rates to identify students who may need support.
2. Check submission timing to understand student work patterns.
3. Monitor late submission trends to adjust deadlines if needed.
4. Identify assignments with low engagement for intervention.

### Managing grading workflows

1. View which assignments need grading assignments.
2. Track grading progress and identify bottlenecks.
3. Ensure timely grade release to students.
4. Balance grading workload across staff members.

### Assignment planning

Use dashboard insights to improve future assignments:

- Analyze submission patterns to optimize due dates.
- Review average scores to calibrate difficulty.
- Identify assignments that need clarification based on submission rates.
- Track time-to-completion for better scheduling.

## Exporting Data

Export assignment data for further analysis, department records, or to analyze assignment effectiveness:

- Submission reports
- Grading summaries
- Student progress data
- Autograder results

## Best Practices

- **Check regularly**: Review the dashboard at least weekly to stay informed.
- **Act on alerts**: Address flagged issues promptly to support student success.
- **Use filters**: Focus on specific assignment types or time periods for targeted analysis.
- **Export data**: Download reports for deeper analysis or record-keeping.
- **Share insights**: Use dashboard data to inform teaching team discussions.
