# Course Settings

Course settings allow instructors to configure global preferences that apply across all course features.

## Time Zone Configuration

Pawtograder supports configurable time zones for displaying dates and times throughout the platform. This ensures that all deadlines, timestamps, and schedules are shown in the appropriate time zone for your course.

### Setting Your Course Time Zone

1. Navigate to **Course Settings** in the course navigation menu
2. Find the **Time Zone** setting
3. Select your preferred time zone from the dropdown menu
4. Click **Save** to apply the changes

Once set, all dates and times throughout Pawtograder will be displayed in this time zone for all users in the course.

### Default Time Zone

If no time zone is configured, Pawtograder defaults to **America/New_York** (Eastern Time).

### Where Time Zones Apply

The configured time zone affects:

- **Assignment due dates**: Deadlines shown to students and staff
- **Survey deadlines**: When surveys close for responses
- **Poll deactivation times**: Automatic poll closing times
- **Office hours schedules**: Queue availability times
- **Gradebook timestamps**: When grades were entered or updated
- **Discussion board posts**: Post and reply timestamps
- **Self-review deadlines**: When self-reviews are due
- **Submission timestamps**: When students submit work
- **Export files**: Timestamps in the configured time zone

### Time Zone Display

When a time zone is configured:

- All users see dates and times in the course's configured time zone
- Countdown timers automatically calculate based on the configured time zone
- Date pickers use the configured time zone for deadline selection
- Export files include timestamps in the configured time zone

### Admin View Default Time Zone

For system administrators viewing courses across multiple time zones, a default time zone can be configured for the admin view to ensure consistent time display when managing multiple courses.

<Note>
The time zone setting is course-wide and applies to all students and staff in the course. Individual users cannot override this setting.
</Note>

<Tip>
Set your time zone to match your institution's location or the primary location where your course is taught. This ensures that due dates and times are intuitive for your students.
</Tip>

## Late Token Policy

Configure the class-wide late token policy that determines how many late tokens students receive and how they can be used for assignment extensions.

### Configuring Late Tokens

1. Navigate to **Course Settings**
2. Find the **Late Tokens** section
3. Set the number of late tokens available to each student
4. Configure how many tokens can be used per assignment (set per-assignment)
5. Define the extension period for each token (typically 24 hours)

See [Due Date Exceptions](/staff/assignments/intro#due-date-exceptions) for more details on late token configuration, and the [Late Tokens](/students/assignments/late-tokens) documentation for how students use late tokens.

## Grading Conflicts

Course staff can declare conflicts of interest to ensure they don't grade assignments where conflicts exist.

Graders can self-input their conflicts under **Course Settings** → **Grading Conflicts**, and instructors can review and enter conflicts for all graders in the class.

See [Grading Assignments](/staff/assignments/grading-assignments) for more details on managing grading conflicts.

## Feature Flags

Feature flags let you enable or disable specific course features. Navigate to **Course Settings** → **Feature flags** to manage them.

Available flags include:

- **Discussion board** — show or hide the discussion board for students and staff
- **Office hours** — show or hide the office hours queue
- **Surveys** — show or hide surveys
- **Polls** — show or hide polls
- **Gradebook What-If** — allow students to use the What-If grade calculator

When a flag is disabled, the corresponding navigation item and dashboard widget are hidden automatically — no page refresh required. Staff can still access the feature directly via URL when a menu entry is hidden.

## GitHub Diagnostics

Instructors can diagnose and fix GitHub account issues for individual students from the **Enrollments** page.

### Accessing diagnostics

1. Navigate to **Course Settings** → **Enrollments**.
2. Click the GitHub icon (purple) next to a student's row.

### What the diagnostics modal shows

- **Linked account** — the GitHub username currently linked to the student's Pawtograder account.
- **Username changed** — whether the student's GitHub username has changed since they linked their account (the stored username vs. the current one).
- **Organization** — whether the student has joined the course GitHub organization (active, pending invitation, or not joined).
- **Student team** — whether the student is a member of the course student team.
- **Database org status** — whether Pawtograder has confirmed the student's org membership.

### Available actions

- **Sync permissions** — re-runs the GitHub sync for the student, updating their username and re-inviting them to the org and team if needed.
- **Unlink GitHub identity** — removes the student's GitHub identity from Pawtograder and removes them from the course GitHub organization. Use this when a student needs to re-link a different GitHub account.
