# Instructor Dashboard

The instructor dashboard provides a comprehensive overview of your course's grading status, student activity, and system health. Access it from the course navigation menu.

## Assignment Grading Overview

The dashboard displays all assignments in a consolidated table organized into three sections:

### Needs Attention

Assignments with incomplete work, open regrade requests, or other items requiring action:

- Assignments with incomplete submission reviews
- Assignments with incomplete review assignments
- Assignments with open regrade requests

### Upcoming with No Blocking Work

Assignments with upcoming due dates that have no pending grading or issues.

### Complete or Stable

Past-due or undated assignments where all grading is complete and no issues remain.

## Dashboard Metrics

For each assignment, you can see:

- **Grading Due**: The due date for grading work (in your course timezone)
- **Submitted Students**: How many students have submitted vs. total enrolled
- **Submission Reviews**: Completion status of overall submission grading (one per submission)
- **Review Assignments**: Individual grading tasks assigned to staff members. Multiple graders can be assigned to review different parts of the same submission, so review assignment counts may exceed submission review counts.
- **Grades Released**: Whether grades have been released (fully, partially, or not at all)
- **Regrades / Extensions**: Open regrade requests and students with active due date extensions

## Summary Cards

At the top of the dashboard, summary cards display:

- **Assignments Needing Attention**: Total count of assignments requiring action
- **Incomplete Submission Reviews**: Total across all assignments
- **Incomplete Review Assignments**: Total grading tasks pending
- **Grade Release Status**: Fully released vs. total releasable assignments
- **Open Regrade Requests**: Total open requests across all assignments

## Quick Actions

Each assignment row provides links to:

- **Grading Dashboard**: Detailed grading progress and rubric completion
- **Submissions**: View all student submissions
- **Regrade Requests**: Manage student regrade requests
- **Due Date Exceptions**: View and manage extensions

## GitHub Diagnostics

Instructors can diagnose and fix GitHub account issues for individual students from the **Manage → Enrollments** page. Click the GitHub icon on any student row to open the diagnostics modal, which shows:

- **Linked account**: The GitHub username currently linked to the student's Pawtograder account.
- **Username changed**: Whether the student's GitHub username has changed since it was last synced (the stored username vs. the current one from the GitHub API).
- **Organization membership**: Whether the student is an active or pending member of the course GitHub organization.
- **Student team membership**: Whether the student is a member of the course's student team.
- **Database org status**: Whether Pawtograder has confirmed the student's org membership.

**Actions available:**

- **Sync permissions** — Forces a re-sync of the student's GitHub username and re-invites them to the org/team/repositories as needed. Bypasses the 24-hour cooldown that applies to student-initiated syncs.
- **Unlink GitHub identity** — Removes the student's GitHub identity from Pawtograder, removes them from the course GitHub organization, and clears stored GitHub metadata. The student will need to re-link their GitHub account.

## Office Hours Activity

View recent help queue activity including:

- Open help requests
- Average wait time
- Total requests resolved in the last 24 hours

## Discussion Board Activity

Monitor discussion board engagement:

- Recent posts and replies
- Unanswered questions
- Most active threads

## Workflow Run Status

Track autograder and background job health:

- **Last Hour**: Recent workflow runs and error count
- **Last 24 Hours**: Daily workflow statistics
- **Recent Errors**: Quick access to failed workflow runs
